# 俄罗斯方块微信小游戏 - 泼墨事件需求规格

## 概述
本需求描述“泼墨事件”的具体实现规格，包括视觉表现、动画流程、渲染层级、游戏交互影响及与事件系统的协作方式。

---

## 事件基本信息

| 属性 | 值 |
|------|-----|
| **事件名称** | 泼墨事件（Ink Splash） |
| **事件 ID** | `1002` |
| **事件类型** | 一次性全屏效果 |
| **触发条件** | 由事件系统调度，满足触发间隔 + 概率判定后触发 |
| **影响范围** | 全屏覆盖，遮挡全部游戏画面（除UI按钮外） |

## 事件注册

- **注册关卡**：第 **7** 关（即关卡 7）。
- **注册方式**：在事件系统配置中，将事件 ID `1002` 加入第 7 关的 `eventIds` 列表。
- **配置示例**：
  ```json
  {
    "eventIds": [1002],
    "triggerBlocks": [5, 8, 10],
    "triggerProbability": 100
  }

## 动画流程
- 泼墨事件触发后，按以下时序执行全屏遮罩动画：
| 阶段 | 持续时间 | 进度范围（t）| 效果描述 | 关键参数变化 |
|------|-----|-----|-----|-----|
| **缩放进入** | 0.7 秒 | 0 → 1/7 | 泼墨纹理从屏幕中心以极小比例匀速放大至完全覆盖屏幕 | scale: 0 → 1.0, alpha: 保持 1.0 |
| **静止展示** | 5 秒 | 1/7 → 6/7 | 泼墨纹理保持全屏覆盖，游戏画面被完全遮挡 | scale: 保持 1.0, alpha: 保持 1.0 |
| **淡出消失** | 2.5 秒 | 6/7 → 1 | 泼墨纹理透明度从完全不透明匀速过渡至完全透明 | scale: 保持 1.0, alpha: 1.0 → 0.0 | 

## 渲染层级
- 泼墨纹理应渲染在游戏画布的最上层，覆盖所有方块、网格等游戏元素。
- UI 按钮（左、右、下、变换、暂停）需保持可交互，可以泼墨遮挡，但是不影响按键的响应。
- 通关或死亡面板在墨汁之下
- 暂停面板在墨汁之上

## 注意事项
- 泼墨事件中，玩家通关，或者死亡，立刻进行谈出消失
- 泼墨事件中，玩家点击暂停，弹出暂停面板，此刻墨汁也应该暂停
