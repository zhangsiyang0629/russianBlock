# 俄罗斯方块微信小游戏 - 无尽模式需求规格

## 概述
本需求描述无尽模式的解锁条件、UI布局调整、游戏规则、排行榜关联及难度递增机制。无尽模式作为关卡模式的进阶玩法，提供持续挑战与分数冲榜体验。无尽模式同样消耗2体力。

---

## UI 布局调整

### 封面按钮新增
- 在封面 **Play 按钮下方** 新增一个按钮：
  - **按钮文案**：`Infinite`（或 `无尽模式`）
  - **按钮样式**：与 Play 按钮风格统一，视觉上略作区分（如不同颜色或边框）。
  - **点击行为**：
    - 若已解锁无尽模式 → 进入无尽模式游戏。
    - 若未解锁 → 弹出气泡提示。

### 隐私授权相关 UI 移位
- 将原有的 **「隐私条款」勾选框** 及相关的 **头像授权按钮** 等 UI 元素，**同步下移**至 `Infinite` 按钮下方。
- 确保主界面布局从上到下依次为：Play 按钮 → Infinite 按钮 → 隐私授权相关 UI。

## 解锁条件

### 解锁规则
- **前置关卡**：玩家必须**通关第 8 关**方可解锁无尽模式。
- **判断依据**：从云数据库 `player_data` 表中读取玩家的 `level` 字段，若 `level > 7`，则视为已解锁。

### 未解锁交互
- 玩家在**未解锁**状态下点击 `Infinite` 按钮时，弹出气泡提示。
- **气泡文案**：`level > 7`
- **气泡样式**：半透明深色背景，白色文字，指向 `Infinite` 按钮，持续约 2 秒后自动消失。
- **气泡位置**：位于 `Infinite` 按钮上方或侧方，避免遮挡按钮本身。

### 解锁后交互
- 玩家已解锁时，点击 `Infinite` 按钮**直接进入无尽模式游戏**，无需二次确认。

## 游戏区域规格

### 区域尺寸
- 无尽模式使用**固定**游戏区域尺寸：**15 列 × 23 行**。
- 该尺寸不受关卡系统影响，始终恒定。

### 界面适配
- 游戏主画布应根据 15×23 的格子尺寸动态调整绘制比例，确保完整显示。
- 方块大小、网格线间距等需相应缩放，保持视觉一致性。

---

## 音乐播放规则

### 选曲逻辑
- 无尽模式下，**所有背景音乐**（`music_1` ~ `music_x`）随机播放。
- **不包含**封面音乐（若有独立的封面主题曲，则排除在外）。
- 当前曲目播放完毕后，**自动随机切换到下一首**，不可重复连续播放同一首。
- 切换时应有简短淡入淡出过渡（约 0.3 秒），避免突兀切换。

### 加载策略
- 复用已有的音乐双源容灾加载逻辑（优先云存储，回退 COS）。
- 无尽模式开始前预加载多首音乐，避免播放中断。

## 排行榜关联

### 分榜规则
| 排行榜 | 数据来源 | 说明 |
|--------|----------|------|
| **分数排行榜** | **仅无尽模式** | 仅记录无尽模式中的最高分，关卡模式分数**不上分数榜**。 |
| **关卡排行榜** | **仅关卡模式** | 仅记录关卡模式中的最高关卡数，无尽模式关卡数**不上关卡榜**。 |

### 数据写入
- 无尽模式游戏结束时，通过云函数将本局分数与 `rankings` 表中的 `highScore` 比较，若更高则更新，并同步更新 `player_data` 表。
- 更新字段：`highScore`、`updateTime`。

---

## 事件系统注册

- 无尽模式注册 **所有已定义的事件**（即 `eventIds: [-1]` 或未配置时的全事件默认行为）。
- 事件触发机制与关卡模式一致，遵循事件系统的配置规则（触发间隔、概率判定等）。
- 事件注册表需保持同步，新增事件自动适用于无尽模式。

## 砖块下落速度递增

### 速度档位

砖块下落速度随分数增加逐步提升，共 **7 个档位**：

| 档位 | 分数阈值 | 速度倍率 |
|------|----------|----------|
| 1 | 0 | 1.0 |
| 2 | 300 | 1.1 |
| 3 | 800 | 1.2 |
| 4 | 1500 | 1.3 |
| 5 | 2500 | 1.4 |
| 6 | 4000 | 1.5 |
| 7 | 6000 | 1.6（上限） |

> **配置化要求**：分数阈值与速度倍率必须支持配置，不可硬编码。建议存储在单独的速度配置表中，方便后续数值调整。

### 配置结构示例
```json
{
  "speedLevels": [
    { "score": 0,    "speed": 1.0 },
    { "score": 300,  "speed": 1.1 },
    { "score": 800,  "speed": 1.2 },
    { "score": 1500, "speed": 1.3 },
    { "score": 2500, "speed": 1.4 },
    { "score": 4000, "speed": 1.5 },
    { "score": 6000, "speed": 1.6 }
  ]
}
```

## 速度切换表现
- 当玩家累计分数首次达到某个档位的分数阈值时，触发速度升级。
- 不能降级：即使分数回退（理论上无尽模式分数只增不减），速度倍率保持已激活的最高档位。

## 速度提升提示效果

### 触发时机
- 每次速度倍率发生变化（如从 1.0 升至 1.1）时，屏幕上显示提示文字。

### 提示文案
- 固定文案：SPEED UP !!!

### 动画效果
1. 淡入：0.3 秒内透明度从 0 过渡到 1。
2. 展示：保持 1 秒。
3. 淡出：0.3 秒内透明度从 1 过渡到 0。
4. 总计：约 1.6 秒。

### 样式要求
- 文字居中于游戏区域中央。
- 字体粗体，较大字号（建议 Canvas 绘制时使用 40~48px）。
- 颜色：明亮醒目的黄色或橙色（如 #FFD700），可加外描边增强可读性。
- 出现时伴随轻微放大缩小（从 0.8 缩放到 1.0），增强视觉冲击力。
