# 俄罗斯方块微信小游戏 - 关卡进入动画需求规格

## 概述
本需求描述每进入新的一关时，游戏开始前播放的过渡动画。动画表现一个说话的人物角色，并根据当前关卡与上一关的难度差异，在对话气泡中展示不同的提示文本。动画播放期间游戏暂停，动画结束后方块才开始下落。

---

## 动画播放规则

### 触发时机
- 每次**进入新的一关**时（包括关卡模式过关后进入下一关、无尽模式开始时），在游戏正式开始前播放该动画。
- 重新开始当前关卡（如死亡后复活）**不播放**该动画。

### 动画时长
- 目标播放时长：**3 秒**。
- 若动画序列帧的实际总时长**不足 3 秒**，则**循环播放**直至满 3 秒。
- 若动画序列帧的实际总时长**超过 3 秒**，则截断至 3 秒（即 3 秒一到立即停止）。

### 动画位置
- 动画整体展示于**游戏区域的左下角**。
- 人物底部对齐游戏区域底边，左侧对齐游戏区域左边，可留少量内边距（如 4~8 像素）。

---

## 对话气泡绘制

### 气泡位置
- 位于动画人物的 **1 点钟方向**（右上方）。
- 气泡与人物头部的距离约为 **1 个游戏格子的宽度**（具体数值根据游戏区域尺寸动态计算）。
- 气泡不得超出游戏区域边界，若超出则自动向左或向下偏移。

### 气泡样式
- **边框**：黑色实线描边，线宽 2~3px。
- **背景**：白色填充。
- **形状**：圆角矩形，圆角半径建议 6~8px。
- **气泡尾巴**（可选）：指向人物嘴部方向的小三角，可省略以简化实现。
- 气泡内边距建议：左右 12px，上下 8px。

### 气泡尺寸
- 气泡宽度自适应文本内容，根据文本长度动态调整，但不超过游戏区域宽度的 70%。
- 气泡高度根据文本行数自适应，单行文本时高度约 30~40px。

---

## 对话内容规则

根据当前关卡与上一关的难度差异，在气泡中展示不同文本：

| 情景 | 判断条件 | 显示文本 |
|------|----------|----------|
| **速度提升** | 当前关卡的下落速度倍率 **>** 上一关的下落速度倍率 | `SPEED UP !!!` |
| **区域变大** | 当前关卡的游戏区域尺寸（列×行）**>** 上一关的区域尺寸，且速度倍率**未提升** | `BIG BIG !!!` |
| **其他情况** | 上述条件均不满足（如速度相同、区域相同，或无尽模式首次进入） | `GOOD LUCK` |

> **判断优先级**：先判断速度提升，再判断区域变大，最后兜底为 `GOOD LUCK`。
> 对于第一关（无上一关），默认显示 `GOOD LUCK`。
> 对于无尽模式首次进入，若无前置关卡参考，显示 `GOOD LUCK`。

### 文本样式
- **字体**：粗体，加粗强调。
- **字号**：建议 16~20px（Canvas 绘制时根据游戏区域动态适配）。
- **颜色**：
  - `SPEED UP !!!`：红色（`#FF3333`），警示感。
  - `BIG BIG !!!`：橙色（`#FF8800`），提醒感。
  - `GOOD LUCK`：绿色（`#33AA33`），鼓励感。
- **对齐**：文本在气泡内居中显示。

## 动画结束与过渡

### 淡出阶段
- 动画播放满 3 秒后，进入淡出阶段。
- **淡出时长**：**0.5 秒**。
- 对话气泡与人物动画**同步淡出**，透明度从 1.0 匀速过渡至 0.0。
- 淡出完成后，移除人物动画和气泡。

### 游戏正式开始
- 淡出阶段结束后（即动画触发后约 3.5 秒），游戏区域**开始生成第一个砖块**，方块开始正常下落。
- 淡出期间游戏暂停，不响应方向键操作。

---

## 渲染层级

### 层级顺序（从底到顶）

1. **游戏背景**（网格线、背景色等）
2. **已落定的方块**（过渡动画时通常为空）
3. **说话人物动画**（序列帧）
4. **对话气泡**（Canvas 绘制，位于人物 1 点钟方向）
5. **UI 按钮层**（操作按钮、暂停按钮等）

### 层级规则
- 对话气泡始终在人物动画**之上**绘制。
- UI 按钮层始终在最顶层，不被动画遮挡。

---

## 游戏交互影响

| 交互项 | 动画播放期间行为 |
|--------|------------------|
| 方块下落与移动 | **暂停**，动画期间不生成新方块。 |
| 暂停按钮 | **可用**。玩家可点击暂停按钮，暂停游戏并同时暂停动画计时。 |
| 其他 UI 按钮 | 正常响应，不受影响。 |
