# 俄罗斯方块微信小游戏 - 炸弹事件需求规格

## 概述
本需求描述“炸弹事件”的具体实现规格，包括视觉表现、动画流程、爆破逻辑及与事件系统的协作方式。

---

## 事件基本信息

| 属性 | 值 |
|------|-----|
| **事件名称** | 炸弹事件（Bomb Blast） |
| **事件 ID** | `1003` |
| **事件类型** | 一次性爆破效果 |
| **触发条件** | 由事件系统调度，满足触发间隔 + 概率判定后触发 |
| **影响范围** | 随机爆破已落定方块 |

## 事件注册

- **注册关卡**：第 **8** 关（即关卡 8）。
- **注册方式**：在事件系统配置中，将事件 ID `1003` 加入第 8 关的 `eventIds` 列表。
- **配置示例**：
  ```json
  {
    "eventIds": [1003],
    "triggerBlocks": [8, 12, 15],
    "triggerProbability": 100
  }

## 触发前置条件
- 已落定砖块层数检查
  - 触发炸弹事件前，必须先检查当前游戏区域中已落定砖块的有效层数。
  - 有效层数定义：从游戏区域底部向上逐行扫描，找到第一行没有任何已落定砖块的行，该行到区域底部的距离即为有效层数。
  - 层数不足时不触发：若有效层数 < 2（即 0 层或 1 层），炸弹事件不触发。事件系统跳过本次触发机会，继续累计砖块计数
  - 层数满足时：正常执行炸弹事件流程。

## 炸弹动画流程

| 阶段 | 持续时间 | 效果描述 |
|--------|------|------|
| 炸弹出现与引线燃烧 | 1 秒 | 在游戏区域随机位置展示炸弹序列帧动画（引线燃烧效果）。若序列帧动画时长不足 1 秒，则循环播放直至满 1 秒 |
| 爆炸缩放 | 0.5 秒 | 炸弹动画消失，同一位置展示 boom.png 图片，从极小比例（如 0.1 倍）匀速放大至原始尺寸。 |
| 爆炸停留并抖动 | 1 秒 | boom.png 保持原始尺寸，并在原地随机抖动。抖动幅度建议 ±3~5 像素，频率约每秒 10~15 次。 |
| 淡出消失 | 1 秒 | boom.png 透明度（alpha）从 1.0 匀速过渡至 0.0，完全消失。 |

## 炸弹位置
- 炸弹大小：占据 2 × 2 个格子的区域。
- 生成位置：在游戏区域内的已落定砖块上方随机选择一个合法位置。炸弹的 2×2 范围必须完全在游戏区域内，不得超出边界。  

## 爆破逻辑
### 触发时机
- 爆破在炸弹动画的爆炸缩放阶段结束时（即动画第 1.5 秒时）执行。即 boom.png 完成放大后，立即执行爆破。

### 爆破范围与数量
- **爆破范围**: 以炸弹中心为基准，向四周扩展形成一个 5 × 5 格子的影响区域。
- **爆破数量**: 在该 5×5 区域内，随机选取 5 ~ 25 个格子（含已落定砖块和空位）。仅爆破其中实际存在已落定砖块的格子，空位不受影响。

### 爆破特效
- 被爆破的砖块复用消层特效（砖块被推倒/破碎 + 灰尘粒子效果）。
- 特效播放方式与普通消层完全一致，确保视觉一致性。

### 爆破后处理
- 被标记的砖块从游戏数据中移除。
- 若爆破导致上方砖块悬空，保持悬空
- 爆破本身不触发计分，消分规则仅由正常消层判定。
