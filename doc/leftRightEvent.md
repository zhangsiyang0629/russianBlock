# 俄罗斯方块微信小游戏 - 左右混乱事件需求规格

## 概述
本需求描述“左右混乱”事件的具体实现规格，包括视觉表现、交互逻辑、生效范围及与其他系统的协作方式。

---

## 事件基本信息

| 属性 | 值 |
|------|-----|
| **事件名称** | 左右混乱（Left-Right Confusion） |
| **事件 ID** | `1001` |
| **事件类型** | 单次生效（仅影响一个方块） |
| **触发条件** | 由事件系统调度，满足触发间隔 + 概率判定后触发 |

---

## 视觉表现

### 特殊方块外观
- 被事件影响的方块与普通方块有明显视觉差异。
- 方块周围或表面**冒绿色烟雾**，烟雾需具备动态效果。
- 具体设计要求：
  - **烟雾颜色**：绿色，建议色值 `#4CAF50` 或 `#8BC34A`，带半透明渐变。
  - **烟雾形态**：从方块表面向上升腾、缭绕，粒子或帧动画均可。
  - **烟雾范围**：不超出方块边界过多，建议烟雾扩散半径不超过方块宽度的 1.5 倍。
  - **方块本体**：保留方块原有颜色，可叠加一层绿色光晕或边缘描边以示区分。

### 特效实现建议
- **方案一（粒子系统）**：在方块上方生成 8~12 个绿色粒子，粒子向上飘散并渐隐，循环生成。
- **方案二（帧动画）**：预制 4~6 帧绿色烟雾动画，在方块上循环播放。
- 特效需在方块落定后立即消失。

### 特效持续时间
- 从方块生成开始，持续至方块**落地锁定**或**本局游戏结束**。

## 交互逻辑

### 方向颠倒规则
- 用户在**控制区**点击方向键时，该特殊方块的移动方向**左右颠倒**：

| 用户点击 | 方块实际移动 |
|----------|--------------|
| 左按钮（`←`） | **向右**移动一格 |
| 右按钮（`→`） | **向左**移动一格 |

### 不受影响的操作
以下操作对该特殊方块**正常工作**，不受方向颠倒影响：
- **向下加速键**（`↓`）：加快方块下落速度。
- **变换键**（`⟳`）：旋转/变换方块朝向。

### 按钮视觉反馈
- 方向按钮的视觉表现**不改变**（按钮图标、位置、颜色等与平常一致）。
- 仅方块的**实际移动逻辑**发生变化。

## 生效范围与时机

### 生效范围
- 该事件**仅影响一个方块**，即事件触发后生成的**下一个**方块。
- 若事件触发时，当前已有方块正在下落，则特殊方块在**当前方块落定后生成**。
- 特殊方块落定后，事件效果结束，后续方块恢复正常。

### 事件标记
- 事件触发后，置一个标记 `nextBlockConfused = true`。
- 下一个方块生成时，检查该标记：
  - 若为 `true`，生成带烟雾特效的特殊方块，并启用方向颠倒逻辑。
  - 方块落定后，将标记重置为 `false`。

### 与其他事件的兼容
- 同一时刻**只有一个事件效果**作用于当前方块。
- 若特殊方块尚未落定时，事件系统又触发了新事件（概率极低，但需处理）：
  - 新事件**覆盖**旧事件效果。`nextBlockConfused` 标记被新事件的标记替换。
  - 正在下落的特殊方块保持当前效果不变（或根据新事件类型决定是否覆盖，此处建议保持当前方块的混乱效果直至落定）。

## 与事件系统的协作

### 触发后的事件系统响应
1. 事件系统判定触发“左右混乱”事件（ID: 1001）。
2. 事件系统调用 `handleConfusionEvent()` 处理函数。
3. 处理函数设置全局标记 `nextBlockConfused = true`。
4. 事件系统记录本次事件已消耗，进入下一轮触发间隔计数。
