# 俄罗斯方块微信小游戏 - 游戏分享需求规格

## 概述
本需求描述游戏中分享功能的交互逻辑与UI表现。在当前未接入正式广告的阶段，将原本的“广告复活”功能暂时替换为“分享复活”功能。
在排行榜、胜利面板等场景中增加分享入口，借助微信社交生态实现游戏传播。

---

## 分享复活功能

### 触发场景
- 玩家游戏**死亡后**（方块堆叠到顶等），弹出游戏结束/死亡面板。
- **体力耗尽**，玩家点击play或infinite按钮就会谈出面板，面板中就一个分享按钮
- **特殊情况**，死亡后，在弹出死亡面板中，此刻玩家体力为0，点击restart不在弹出分享按钮，直接弹汽包：体力不足。

### UI表现

#### 未接入正式广告时（当前阶段）
- 死亡面板中显示 **「分享复活」** 按钮，替换原本的“看广告复活”按钮。
- **按钮文案**：分享图标 + `复活`
  - 左侧显示分享图标（可使用微信官方分享图标或自定义），右侧显示文字“复活”。
- **按钮样式**：与死亡面板整体风格统一，保持与原本广告按钮相同的尺寸和位置。

#### 接入正式广告后（未来阶段）
- 死亡面板中恢复显示 **「看广告复活」** 按钮。
- 可保留分享按钮作为额外入口（非强制）。

### 交互逻辑
1. 玩家点击「分享复活」按钮。
2. 调用微信分享接口（`wx.shareAppMessage` 或 `wx.onShareAppMessage`），拉起分享面板。
3. 玩家可选择：
   - **分享到朋友圈**
   - **发送给朋友**
4. 完成分享操作后（玩家完成分享动作并返回游戏），触发**复活逻辑**：
   - 玩家在当前游戏状态原地复活。
   - 体力值不额外扣除。
   - 移除已落定的方块。
5. 若玩家取消分享（未实际分享），不触发复活，返回死亡面板。

### 分享内容配置
```javascript
wx.shareAppMessage({
  title: '快来挑战我的最高分！', // 可自定义
  imageUrl: '', // 分享卡片图片，建议为游戏截图或Logo
  query: '', // 可携带参数，如邀请码等
  success: () => {
    // 分享成功，执行复活逻辑
    revivePlayer();
  },
  fail: () => {
    // 分享失败或取消，不复活
    console.log('分享取消');
  }
});
```

## 排行榜分享功能

### UI表现
- 在 排行榜界面 的 右上角 添加一个分享图标按钮。
- 按钮样式：仅显示分享图标（无需文字），图标风格与排行榜面板整体一致。
- 按钮尺寸：建议 32×32 ~ 44×44 像素（可点击区域规范）。

### 交互逻辑
1. 玩家在排行榜界面点击右上角分享按钮。
2. 调用微信分享接口，拉起分享面板。
3. 玩家可选择分享到朋友圈或发送给朋友。
4. 分享内容可包含当前榜单类型（分数榜/关卡榜）及玩家自身排名等信息。
5. 分享完成后无额外游戏内奖励，仅作为传播入口。

### 分享内容配置
```javascript
wx.shareAppMessage({
  title: '我通关了第X关，得分XXX，来挑战我吧！',
  imageUrl: '', // 通关截图或游戏Logo
});
```

## 胜利面板分享功能

### UI表现
- 在游戏 胜利通关面板 上，新增一个分享按钮。
- 按钮文案：仅显示「分享图标」（无需文字）。
- 按钮位置：胜利面板内，可放在「下一关」按钮旁或面板右上角，避免遮挡主要操作。
- 按钮样式：与面板整体风格统一。

### 交互逻辑
1. 玩家在胜利面板点击分享按钮。
2. 调用微信分享接口，拉起分享面板。
3. 玩家完成分享操作后，返回胜利通过面板，不自动跳转下一关。
4. 玩家可继续点击「下一关」按钮进入后续流程。
5. 分享内容可包含当前通关关卡数、得分等信息。

### 分享内容配置
```javascript
wx.shareAppMessage({
  title: '我刚刚通关了第X关，来挑战我吧！',
  imageUrl: '', // 通关截图或游戏Logo
});
```

## 全局分享配置

### 默认分享内容
- 当玩家未进行上述特定分享操作，而是在游戏内通过右上角胶囊按钮或全局菜单触发分享时，使用默认分享配置：
```javascript
wx.onShareAppMessage(() => ({
  title: '一款好玩的俄罗斯方块小游戏，快来挑战！',
  imageUrl: '', // 游戏Logo
}));
```
